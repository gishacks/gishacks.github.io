## مرحله اول تحلیل مطلوبیت: تعیین معیارها

معیارهای مورد استفاده:
- تراکم جمعیت  
- فاصله از مدارس موجود  
- فاصله از گسل *  
- شیب *  
- آلودگی هوا  
- فاصله از پارک *  

برای معیارهای *دار در **Model Builder** شاخه‌های جداگانه ساخته می‌شود:
- یک شاخه برای شیب 
- یک شاخه برای فاصله از گسل

مدل مورد استفاده همان مدل جلسه قبل است.  
در مسیر تحلیل:
- تا محاسبه **Slope** پیش می‌رویم
- سپس ابزار **Euclidean Distance** اجرا می‌شود

## تنظیمات محیطی (Environment Settings)

بهتر است تنظیمات محیطی برای کل مدل یکسان در نظر گرفته شود،  
نه اینکه در هر ابزار (مثل Euclidean Distance) جداگانه تنظیم شود.

مسیر:
Model Builder > Environment > Processing Extent
- انتخاب گزینه دوم
- می‌خواهیم شبیه مناطق شهرداری باشد
- در بخش **Raster Analysis**:
  - **Cell Size** ثابت (مثلاً ۱۰۰ متر)

نکات:
- در تحلیل‌های رستری، تعیین **Cell Size** و **Extent** الزامی است
- هر تحلیل می‌تواند تنظیمات مستقل داشته باشد
- اما در این پروژه بهتر است یکسان باشند، چون خروجی‌ها بعداً با هم ترکیب می‌شوند
- Cell Size پیشنهادی: ۱۰۰ متر

## مرحله دوم تحلیل مطلوبیت: امتیازدهی (Rating)

می‌توان مقیاس امتیازدهی را به‌صورت دلخواه تعریف کرد.  
مثال:
- مقیاس از **-۵ تا +۵**

نکته مهم:
- وقتی می‌گوییم **-۵ نامطلوب‌ترین**، منظور **نامطلوب‌ترین حالت ممکن** است، نه وضعیت موجود

 امتیازدهی:
- فاصله از گسل:
  - بیشترین فاصله → **+۵**
  - کمترین فاصله → **-۵**
- شیب:
  - کمترین شیب → **+۵**
  - بیشترین شیب → **-۵**

---

## ابزار امتیازدهی

برای امتیازدهی از ابزار **Reclassify** استفاده می‌شود.  
به ازای هر معیار، یک **Reclassify** داریم.

## اجرای مدل و رفع خطا

در صورت اجرا نشدن مدل:

- داده رستری را **Project** می‌کنیم:
- Project > Input Raster > Output1
  - سیستم مختصات: **32639**

- تبدیل داده توپوگرافی به رستر **Topo to raster**:
- Feature Layer: `RasterT_output1`

- Type: `Point Elevation`
- Run

## Reclassify خروجی‌ها

- یک بار خروجی **Slope**
- یک بار خروجی **Euclidean Distance**

مراحل:
1. باز کردن پنجره **Reclassify**
2. انتخاب گزینه **Classify**
3. مشاهده طبقات پیشنهادی
4. انتخاب نوع دسته‌بندی (مثلاً Standard)
5. امکان تعریف دستی کلاس‌ها وجود دارد
6. با زدن OK، جدول امتیازدهی باز می‌شود
- با خود دسته‌بندی‌ها کاری نداریم
- فقط ستون **New Value** را پر می‌کنیم
- مشخص می‌کنیم هر کلاس چه امتیازی دارد

در شیب چون امتیازدهی معکوس است:
- مقادیر بالاتر → امتیاز کمتر
- مقادیر پایین‌تر → امتیاز بیشتر
- باکس آخر معمولاً بدون تغییر باقی می‌ماند

---

بعد از Reclassify:
- داده‌ها دیگر واحد واقعی ندارند
- فقط نشان‌دهنده مطلوب یا نامطلوب بودن هستند

در این مرحله:
- دیگر فقط اپراتور GIS نیستیم
- باید دانش شهرسازی را به کار بگیریم

مثال:
- برای مدرسه ابتدایی:
  - شیب کم اهمیت دارد
  - فاصله از گسل اهمیت بالایی دارد
  - نزدیکی به پارک یک مزیت محسوب می‌شود
